Ping Command Overview

The ping command in Linux is used to test the connectivity between your system and a remote host:

  • Sends ICMP Echo Requests to the target.
  • Measures round-trip time for messages.
  • Helps diagnose network issues.

Basic Usage

Common Commands:

  • ping [hostname/IP]: Sends echo requests to the specified target.
  • ping -c [count] [target]: Limits the number of requests sent.
  • ping -i [interval] [target]: Sets the interval between packets (default is 1 second).
  • ping -q [target]: Displays minimal output (summary only).

Advanced Options

Utilize advanced features of the ping command:

  • ping -s [size] [target]: Specifies the size of the packet in bytes.
  • ping -w [timeout] [target]: Sets the time limit for the ping command.
  • ping -f [target]: Floods the network with packets (requires root privileges).
  • ping -l [preload] [target]: Preloads packets before sending.

Interpreting Results

The ping command provides detailed statistics:

  • Time: Round-trip time for packets in milliseconds.
  • Packet Loss: Percentage of lost packets during the test.
  • TTL: Time To Live value, indicating the number of hops a packet can take.
  • IP Address: Resolved IP address of the target.
