SSH Command Overview

The ssh command in Linux is used to securely connect to remote machines:

  • Enables secure encrypted communication.
  • Supports remote login and command execution.
  • Utilizes public-key cryptography for authentication.

Basic Usage

Common Commands:

  • ssh user@host: Connects to the specified host as the given user.
  • ssh -i [identity_file] user@host: Uses a specific private key for authentication.
  • ssh -p [port] user@host: Specifies a non-default port for connection.
  • ssh [host] [command]: Executes a command on the remote host.

Advanced Options

Utilize advanced features of the ssh command:

  • ssh -N -L [local_port]:[remote_host]:[remote_port] user@host: Sets up port forwarding.
  • ssh -v user@host: Enables verbose output for troubleshooting.
  • ssh-copy-id user@host: Copies local public key to the remote host.
  • ssh-agent: Manages private keys for automated authentication.

Security Features

The ssh command ensures secure communication:

  • Utilizes strong encryption protocols (e.g., AES, RSA).
  • Supports two-factor authentication for added security.
  • Allows host key verification to prevent MITM attacks.
  • Supports tunneling for secure data transfer.
