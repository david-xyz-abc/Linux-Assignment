SCP Command Overview

The scp command in Linux is used for securely copying files between hosts:

  • Utilizes SSH for encrypted transfers.
  • Supports copying files to and from remote systems.
  • Maintains file permissions and metadata.

Basic Usage

Common Commands:

  • scp [file] user@host:[path]: Copies a file to a remote system.
  • scp user@host:[file] [path]: Copies a file from a remote system.
  • scp -r [directory] user@host:[path]: Recursively copies a directory to a remote system.
  • scp -P [port] [file] user@host:[path]: Specifies a custom port for transfer.

Advanced Options

Utilize advanced features of the scp command:

  • scp -C [file] user@host:[path]: Enables compression during transfer.
  • scp -i [identity_file] [file] user@host:[path]: Uses a specific private key for authentication.
  • scp -l [limit] [file] user@host:[path]: Limits the transfer bandwidth in Kbps.
  • scp -o [option] [file] user@host:[path]: Passes additional SSH options.

Security Features

The scp command ensures secure file transfers:

  • Encrypts both data and authentication credentials.
  • Supports public key authentication for secure access.
  • Verifies host authenticity to prevent MITM attacks.
  • Uses SSH configurations for added flexibility and security.
