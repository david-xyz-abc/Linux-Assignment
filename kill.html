Kill Command Overview

The kill command in Linux is used to terminate processes by sending signals:

  • Default signal sent is SIGTERM (terminate gracefully).
  • Requires a process ID (PID) to target specific processes.
  • Can send various signals for specific actions.

Basic Usage

Common Commands:

  • kill PID: Sends SIGTERM to the specified process.
  • kill -9 PID: Forces termination using SIGKILL.
  • kill -l: Lists all available signals.
  • kill -s SIGNAL PID: Sends a specific signal to the process.

Common Signals

The kill command supports various signals:

  • SIGTERM (15): Requests graceful termination.
  • SIGKILL (9): Forces immediate termination.
  • SIGHUP (1): Restarts processes without terminating them.
  • SIGSTOP: Pauses a process without terminating it.

Advanced Features

Utilize the kill command for process control:

  • Combine with ps or top to identify PIDs.
  • Use killall to terminate multiple processes by name.
  • Script automation to manage multiple processes.
  • Ensure proper permissions to avoid Operation not permitted errors.
